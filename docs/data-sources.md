审查日期：2026-07-10

## 重复新闻处理

系统使用两层判断：

1. 规范化网址，移除常见追踪参数、统一主机名和路径；
2. 对标题做清洗和近似匹配，合并跨来源或同一来源的轻微改写。

数据库仍以内容指纹设置唯一约束，防止并发任务在最后一步写入重复记录。批量写入只提交一次事务，避免逐条提交造成锁竞争。

## 独立调度与故障隔离

每个来源拥有自己的启停开关和轮询间隔。某个来源失败时只对该来源指数退避，不会拖住其他来源。`GET /api/news/sources` 返回：

- 是否启用、是否已配置密钥；
- 最近尝试和最近成功时间；
- 抓取数量、写入数量和重复数量；
- 最近错误与下一次尝试时间。

错误日志会清理网址查询参数和已知密钥，避免第三方错误把凭据带入日志。

## 其他数据

- 市场行情：来自雅虎财经（Yahoo Finance）客户端。结果带来源和时间；失败时返回空值，不生成模拟数字。
- 经济日历：保留最近一次有效结果，但超过 7 天便不再展示；模型分析缓存键包含事件内容和模型设置，避免配置变化后误用旧分析。
- X 市场情景：由模型生成，默认每 6 小时更新一次。界面和接口均按“模型推演”标注，不声称来自实时社交平台。

## 配置示例

所有开关和间隔都在 `.env.example` 中列出。修改 `.env` 后重启后端即可重新建立调度任务：

```bash
docker compose restart backend
```
