# 数据源与时间边界

新闻源及轮询间隔统一写在 `config/personal.toml`。Finnhub、Massive、NewsAPI 和 GNews 需要环境变量密钥；Google News RSS 与 Seeking Alpha RSS 不需要密钥。缺少密钥的来源记为 `not_configured`，不会影响其他来源调度。

新闻先按内容哈希去重，再按同日规范化标题做近似去重。跨来源重复新闻不会生成第二条正文记录，但新来源携带的原生 Ticker 会合并到现有记录，并产生新的本地变化序号。系统不验证、推断或补造 Ticker。

每个来源独立退避。来源健康分别记录最近尝试、最近成功、持久化水位、原始数量、写入数量、重复数量和错误代码。上游请求成功但数据库写入失败时，持久化水位不会前进。

经济日历只由调度任务访问上游。内部读取接口始终查询数据库，不会因缓存过期而触发网络请求。上游失败时可使用七天以内的最后一次成功缓存，并标为陈旧。
